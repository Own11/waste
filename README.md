# BAHANDI.ACT — Автоматизация списаний на торговых точках ☕🍰

**MVP-система автоматизации и контроля списаний на торговых точках (Кейс Mentoria Hackathon).**

Проект решает проблему рутинного и неконтролируемого списания продукции сотрудниками заведений, автоматизируя процесс согласования заявок и их выгрузки в учетную систему **iiko**.

---

## 🛠 Стек технологий

* **Backend**: Django 6.0 + Django REST Framework (DRF)
* **Database**: SQLite (минималистично, быстро, не требует настройки СУБД)
* **Frontend**: Mobile Web SPA (Single Page Application) на чистом HTML5 / CSS3 (Dark Theme / Engineering Minimalism) / Vanilla JS (Fetch API). *Никаких тяжелых фреймворков и сторонних сборщиков — быстрая загрузка на любых мобильных устройствах.*
* **Auth**: Token Authentication (DRF) с разделением прав по ролям на уровне API.

---

## 📐 Архитектура и бизнес-логика

В системе определены две ключевые роли пользователей:
1. **Сотрудник (Sender)**: Создает заявки на списания с торговой точки, прикрепляя фото дефекта/продукта и комментарий (длиной не менее 10 символов).
2. **Проверяющий (Checker)**: Контролирует поток заявок. При одобрении заявки система автоматически отправляет данные в сервис интеграции `IikoService`, списывает продукцию в iiko и сохраняет сгенерированный ID акта списания.

```
[Сотрудник (Sender)] ──(Создает заявку с фото)──> [Django DB (on_review)]
                                                         │
[Проверяющий (Checker)] ──(Одобряет заявку)──────────────┼──> [IikoService (Mock API)]
                                                         │            │
                                                      (Успех)      (Задержка 0.5с)
                                                         │            │
[Статус: Списано в iiko] <──(Акт списания сохранен)──────┴────────────┘
```

## 📱 Особенности интерфейса (Mobile Web)

Интерфейс спроектирован в стиле **инженерного минимализма**:
* **Тёмная тема (Dark Mode)** и высокая контрастность (удобно для работы в условиях кофейни/ресторана).
* **Сверхбыстрая скорость**: чистый CSS без использования Tailwind или Bootstrap.
* **Шрифты**: Смесь `Inter` (для читаемости интерфейса) и `JetBrains Mono` (для вывода логов, статусов и технических ID актов).
* **Интерактивность**:
  * Динамическое отображение поля выбора сотрудника для удержания (показывается только при типе списания «С удержанием»).
  * Живой валидатор символов комментария (кнопка отправки заблокирована, пока комментарий короче 10 символов).
  * Интегрированный Lightbox для увеличения фотографии списания в один клик.
  * Тосты с микро-анимацией для мгновенного фидбека по действиям.

## 🚀 Быстрый старт

### 1. Подготовка окружения
Убедитесь, что у вас установлен Python 3.10+.
Склонируйте репозиторий и перейдите в папку проекта:

```bash
git clone https://github.com/Own11/waste.git
cd waste
```

### 2. Виртуальное окружение и зависимости
Активируйте встроенное виртуальное окружение и установите зависимости:

```bash
# Для macOS / Linux:
source venv/bin/activate

# Установка зависимостей (если требуется обновить/доустановить):
pip install -r requirements.txt
```

### 3. База данных и фикстуры
Проект поставляется с готовой базой данных SQLite. Для повторного наполнения демо-данными (торговые точки, аккаунты) выполните:
```python
python manage.py seed_data
```

### 4. Запуск сервера разработки
Запустите Django-сервер:
```python
python manage.py runserver
```
Приложение будет доступно по адресу: **[http://127.0.0.1:8000/](http://127.0.0.1:8000/)**

## 📡 Спецификация API (Endpoints)

| Метод | Эндпоинт | Описание | Доступ |
| :--- | :--- | :--- | :--- |
| **POST** | `/api/token/` | Получение токена авторизации и роли пользователя | Любой |
| **GET** | `/api/outlets/` | Получение списка торговых точек для выбора | Авторизованные |
| **GET** | `/api/senders/` | Получение списка сотрудников для удержания | Авторизованные |
| **POST** | `/api/write-offs/` | Создание новой заявки на списание (multipart data + фото) | Только `sender` |
| **GET** | `/api/requests/` | Получение списка активных заявок со статусом "На проверке" | Только `checker` |
| **POST** | `/api/requests/{id}/review/` | Одобрение (`approve`) или отклонение (`reject`) заявки | Только `checker` |
